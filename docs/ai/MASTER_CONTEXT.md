Premium, editorial portfolio website for **Estúdio Monteiro**, an architecture studio. The site is a multi-page editorial experience built on an exposed hairline grid system ("the architectural plan as language"). Goal: project authority and convert high-end clients into direct contact. Content is being structured from scratch with clear placeholders.

- **Primary proof of authority:** the projects themselves (work speaks; minimal copy).

## Primary Goal
Convey authority and lead the high-end visitor to **direct contact (WhatsApp / form)**. Authority first; the CTA is a quiet consequence of the proof, never pressure.

## User Journey Summary
Home (impact + curation) → Projects (proof) → Project detail (desire) → Studio/About (trust) → Contact (action). A discreet contact CTA is present site-wide.

## Approved Architecture
Multi-page editorial: **Home**, **Projects (index)**, **Project (detail)**, **Studio/About**, **Contact**. See `SITE_ARCHITECTURE.md`.

## Visual Direction
Minimalist, sophisticated, architectural, editorial, silent, premium. Generous whitespace, no template feel. **Signature element: an exposed hairline (1px) grid** used as structure — margins, exposed columns, baseline grid, ritmo tipográfico. Near-monochrome palette (paper off-white, graphite/black, one warm neutral); color comes from project photography. Editorial type pairing (serif display + clean grotesque). Photography is the protagonist.

## Motion Direction
Silent and precise. Lines that *draw themselves* (stroke-draw) on section entry; hairlines with subtle scroll reaction; image reveals via mask/clip; smooth page transitions. Never bouncy or loud. Respect `prefers-reduced-motion`.

## Technical Direction
Next.js (App Router) + TypeScript, Tailwind CSS, GSAP (ScrollTrigger / DrawSVG-style line-draw) and/or Framer Motion, Lenis for smooth scroll. Project content in structured MDX/JSON. Deploy on Vercel. `next/image` for optimized imagery.

Protect Core Web Vitals (LCP/CLS/INP), accessibility (semantics, keyboard, focus, reduced motion), SEO basics, fully responsive. See `QA_CHECKLIST.md`.
